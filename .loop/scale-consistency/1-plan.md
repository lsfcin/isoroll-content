## Plan Review (adversarial, assume small executors)
- Cross-projection `s` comparability could confuse an executor (is iso `s` the same unit as grid `cell_s`?). Verified in code: iso maps model→screen as `ox + x·s` where `x` uses unit coeffs `_UX=_UZ=1`, so a unit voxel edge = `s` px along principal axes — identical unit to `cell_s` (px/voxel) in square/flat grids. Taking `min` across them is well-defined. → Stated explicitly in T2 note so the executor does not "normalize" them.
- `content_extent` MUST mirror the exact orientation→arg mapping already in `matrix.draw_panel` (N/S: cols=w rows=d+h top_rows=d; W/E: cols=d rows=w+h top_rows=w; TOP: w×d) and the MIN_THICK substitution for zero dims, else fit/QC diverge from the drawing. → Fix applied: T2 done-when ties `content_extent` to each draw fn's own autofit basis, so any mismatch fails T2, not silently later.
- "byte-identical" (Carry C1 / clarify) is brittle across PIL PNG chunk variance. → Resolved: all legacy-parity checks use pixel-identical comparison (`ImageChops.difference(...).getbbox() is None`), which is the testable form of the intent. Goldens captured in T1 from unmodified base code so a real reference exists.
- `render_cells` return signature change `(img)→(img, scale_info)` is a breaking internal API change. Grep confirms only 2 in-repo callers (make_tile_guide.py:70, matrix.py:138); no external/test caller. Breaking default output is INTENDED per Carry. → T3 done-when requires updating both callers; Loop 2 grounding re-verifies no third caller appeared.
- CAPTION cells and blank (row,col) have no geometry → must be excluded from the `min` and from the sidecar panels list, or `s` collapses / bbox is meaningless. → Encoded in T3/T4 ("non-blank non-CAPTION").
- C3 "same object across view panels": in `make_tile_guide` layouts all panels share one (w,d,h), so the per-panel-vs-global-`px_per_voxel` invariant in T6 both satisfies C3 and directly validates C1; general `tile_guide_matrix` sheets with heterogeneous objects still pass because every panel is checked against the single sheet `s`. → No grouping-by-object needed; noted so the executor doesn't over-build.
- `sheet_qc.py` importing `tile_guide_render` requires it on `sys.path`; conftest adds src/pipeline for tests, but the CLI `__main__` runs standalone. → T6 note: `__main__` must add `src/pipeline` to path (or use a path-insert like conftest) before importing `content_extent`; flagged for the executor.
